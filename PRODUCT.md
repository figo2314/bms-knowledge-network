# Product

## Register

product

## Users

Primary user is the owner: a senior ELV, BMS, and control-systems practitioner who wants to capture, review, reorganize, and present professional knowledge visually. Secondary users are employers, clients, colleagues, students, and junior engineers who need to understand BMS architecture, field networks, protocols, device roles, and chiller plant control logic.

Users may not be programmers. Editing must feel direct, visual, and forgiving: drag nodes, connect devices, assign protocol and brand metadata, inspect details, and save without touching code.

## Product Purpose

This product is a public-facing personal knowledge network for BMS and ELV systems. It should function as a professional portfolio, a teaching reference, and a living review system. Success means a visitor can quickly see the author's depth of understanding, while the author can continuously add devices, brands, protocols, diagrams, and control logic through an intuitive interface.

Core product areas:

- Spatial BMS topology from server to network controller, FLN, field controller, field device, sensor, and actuator.
- Drag-and-connect graph editing for hierarchy, protocol relationships, and equipment dependencies.
- Multi-brand equipment knowledge library covering vendors, device families, protocols, I/O, wiring notes, and typical use cases.
- Chiller plant learning module with a 3D simulation of water flow, plant components, sensors, actuators, and operating modes.
- Power management and data engineering layer covering meter collection, zone kVA peak, kWh consumption, trend collection, SQL Server storage, database retention, and reporting.
- Public showcase layer that presents the author's system thinking, engineering taste, and practical experience.

## Brand Personality

Expert, spatial, precise, high-end, interactive.

The interface should feel like a refined control room and a premium engineering portfolio at the same time: technical enough for professionals, clear enough for learners, and visually memorable enough to represent the owner's personal brand.

## Anti-references

- Plain wiki pages with long text and no system context.
- Generic dashboard templates with disconnected cards.
- Overly academic diagrams that explain terms but not relationships.
- Marketing landing pages that look impressive but do not let the user work.
- Diagram tools that require programming or complex setup before simple edits.

## Design Principles

1. Show relationships before definitions.
2. Make every concept editable through direct manipulation.
3. Teach through signal flow, water flow, control flow, and failure modes.
4. Support multiple brands without making any one vendor the default truth.
5. Let the product itself demonstrate professional depth.

## Accessibility & Inclusion

Target WCAG AA contrast for public pages and editing surfaces. Support keyboard focus states, readable text at dense UI sizes, reduced-motion preferences, and non-color-only status indicators. 3D and motion features should have static fallback explanations so the content remains understandable without animation.
